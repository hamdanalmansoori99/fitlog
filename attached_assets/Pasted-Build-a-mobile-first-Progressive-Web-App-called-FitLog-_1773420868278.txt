Build a mobile-first Progressive Web App called "FitLog".

Goal:
Create a fitness tracking app that is installable on phones, supports authentication, and stores each user’s private data securely in a database. Every user must only see their own data.

Tech requirements:
- Use a modern full-stack web stack suitable for Replit
- Recommended: Next.js + TypeScript + Tailwind CSS + Prisma + PostgreSQL
- Use authentication for signup/login/logout
- Use a real database for persistent storage
- Make the app a true PWA:
  - installable on mobile
  - responsive
  - app icon
  - manifest
  - service worker / offline-friendly basics
- Build clean reusable components
- Use strong typing and clean folder structure
- Seed sample data for testing
- Add comments where helpful

Design system:
- Dark mode by default
- Support light mode toggle in settings
- Color palette:
  - background: #0f0f1a
  - card backgrounds: #1a1a2e
  - primary accent: #00e676
  - secondary accent: #448aff
  - text: #f5f5f5
  - muted text: #9e9e9e
  - danger/delete: #ff5252
- Typography:
  - clean sans-serif font like Inter or system stack
  - bold headings, regular body text
- Cards:
  - rounded corners 16px
  - subtle shadows
  - slight transparency
- Icons:
  - use Lucide or similar consistently
- UI:
  - bottom navigation bar with 5 tabs:
    1. Home
    2. Workouts
    3. Meals
    4. Progress
    5. Profile
- Add smooth page transitions and subtle micro-animations
- Buttons must have hover and active states
- Primary buttons use green accent
- Empty states should be friendly and helpful, for example:
  - "No workouts yet — tap + to get started!"

Core database models:
Create proper schemas/models for at least:
- User
- Profile
- Workout
- WorkoutExercise
- WorkoutSet
- Meal
- MealFoodItem
- Equipment
- BodyMeasurement
- Settings
- ActivityStat cache or computed summary if useful

Authentication and privacy:
- Every user has private data
- Enforce user-level access control on all routes and queries
- Do not expose another user’s data
- Protect all create/read/update/delete actions
- Include delete account functionality
